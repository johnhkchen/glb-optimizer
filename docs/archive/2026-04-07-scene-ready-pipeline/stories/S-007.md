## Lighting and Weather Presets

### Context

Today, lighting is a mix of hardcoded directional lights and a half-broken reference-image calibration. The user has no way to ask for "midday sun" or "overcast" or "golden hour" — those are the actual concepts a landscape designer thinks in, and they should be the actual controls.

This story consolidates lighting into a small set of named presets that drive **both the bake and the live preview consistently**. Pick "midday sun" once, and the bake lights, the env map, and the preview scene all use it. Reference image calibration becomes one option among many under this umbrella, not a special button.

### Acceptance Criteria

- New `LightingPreset` enum with at minimum:
  - `midday-sun` — bright neutral white, strong key from above
  - `overcast` — soft, low-contrast, slightly cool
  - `golden-hour` — warm orange key, long shadow direction (still rotationally symmetric for bake omni — directional warmth, no horizontal bias)
  - `dusk` — cool blue ambient, low key intensity
  - `indoor` — neutral fill, no strong key, softer
  - `from-reference-image` — uses the reference image calibration system (palette extraction + tinting)
- Preset definitions stored in `static/presets/lighting.js` (or equivalent) with:
  - Bake config: ambient, hemisphere sky/ground, key intensity, fill intensity, env map gradient stops
  - Preview config: same parameters applied to the live scene
  - Tone mapping exposure
- Preset dropdown in the tuning UI panel
- Switching preset updates both bake settings and live preview lights immediately
- Preview lights always match bake lights — consistency is enforced
- Reference image calibration ("from-reference-image" preset) inherits the existing palette extraction work but is now reachable as a setting, not a one-shot button
- Switching preset emits an analytics event
- Per-asset preset choice persists in settings (from S-002)
- Each preset has a one-line description shown in the dropdown ("Bright neutral key light, like noon")

### Out of Scope

- Custom user-authored presets (use case for a follow-up; for now, presets are baked into code)
- HDR environment map upload
- Time-of-day animation
- Per-light editing (the preset is the unit of control)
- Sky-as-background rendering (presets affect lighting only, not the canvas background color)

### Notes

- Each preset is a "named setting bundle." Conceptually similar to profiles from S-003 but baked-in and curated, vs. user-created.
- Get the preset names right — they're user-facing and they should map to terms a designer uses. "Midday sun" not "high-intensity-key-omnidirectional".
- The bake-preview consistency requirement is the main technical lift. Today the bake and live preview have separate light setups with different intensities — this story collapses that into a single preset that applies to both.
