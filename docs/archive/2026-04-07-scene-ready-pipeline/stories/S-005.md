## Bake Tuning Controls — Restore Lost Quality

### Context

The bake pipeline is sound (right price, wrong flavor). Several specific quality issues exist today, all of which need to become tunable rather than being hardcoded compromises:

1. **Dome curvature is gone.** When we adopted vertex-quantile slice boundaries, the umbrella/dome shape we had earlier was lost. Stacked slices now look like flat pancakes rather than a natural plant silhouette.
2. **Ground clipping.** The bottom slice's actual floor sits below Y=0 in the world, which clips into the ground in scene previews.
3. **Center-of-mass drag.** Vertex-quantile slicing places more boundaries near the dense lower foliage, which is correct for vertex distribution but pulls the visual top of the plant down. Plants look squashed.
4. **Over-lit / under-shaded bakes.** The current omni lighting is too uniform and washes out the contrast between leaves.

This story doesn't rewrite the algorithm. It exposes the right knobs in the settings UI from S-002, sets sensible defaults that fix the worst issues, and instruments every control with S-003 analytics so we can learn which defaults actually work.

### Acceptance Criteria

- New settings exposed in the tuning panel:
  - **Dome height factor** (0.0 = flat, 1.0 = full dome) — restores umbrella curvature
  - **Slice distribution mode** (`equal-height` / `vertex-quantile` / `visual-density`) — addresses the center-of-mass drag
  - **Slice count** (overrides the auto adaptive count from earlier work)
  - **Ground alignment** (toggle: snap bottom slice to Y=0)
  - **Bake exposure** (tone mapping exposure 0.5-2.0)
  - **Ambient intensity**
  - **Hemisphere intensity**
  - **Key light intensity**
  - **Bottom fill intensity**
  - **Env map intensity**
  - **Color calibration mode** (none / reference image / preset — full preset enum lands in S-007)
- New "visual-density" slice distribution mode: weights vertex distribution by something other than raw count (e.g., distance from stem axis, or excluding bottom-Y vertices that are part of the trunk) so the boundaries reflect *visible* foliage rather than total geometry
- Ground alignment fix: bottom slice's lowest Y vertex sits exactly at Y=0
- Default values produce a noticeably better-shaded bake than current — leaf-to-leaf contrast visible without going pitch black
- Every control changing fires a `setting_changed` analytics event (free via S-003)
- Each control has inline help text explaining what it affects

### Out of Scope

- Algorithmic improvements beyond exposing existing parameters
- Lighting presets (S-007)
- Reference image calibration improvements (S-007)
- Per-slice customization (one-off tuning per slice)
- Real-time preview as you drag sliders (regenerate-on-change is fine)

### Notes

- The "visual-density" mode is the most interesting technical work in this story. Worth a small spike to validate the approach before implementing.
- Don't try to make any single default perfect. The whole point of this story is that defaults are starting points and humans tune from there. Analytics will tell us what defaults work.
- Pair with S-003 — every slider is also an event source.
