## Scene Preview Realism

### Context

The current 100x stress test puts identical instances on a perfect grid with identical Y rotations. It tells us about FPS but not whether the asset will look right in a planted bed. The user can't visually evaluate "is this asset designer-quality" from a uniform grid.

This story replaces (or augments) the grid stress test with a scene preview that simulates what the asset would look like in realistic placements: scattered, varied in size, oriented per the asset's shape category from S-004.

### Acceptance Criteria

- New "Scene preview" mode in the preview panel, alongside the existing stress test
- **Scene templates** the user can pick from:
  - `hedge-row` — single row, evenly spaced, fixed orientation (tests directional assets)
  - `mixed-bed` — natural scatter with size variation (tests round bushes)
  - `rock-garden` — sparse, larger size variation
  - `container` — small group, tightly clustered
  - `grid` — current behavior, kept for FPS benchmarking
- **Per-instance variation**:
  - Scale: ±20% configurable range
  - Position jitter: configurable amount
  - Y rotation strategy driven by the asset's shape category from S-004:
    - `round-bush` → random Y rotation
    - `directional` → fixed orientation, optional small jitter (±5°)
    - `tall-narrow` → random Y rotation
    - `planar` → aligned to row direction
    - `hard-surface` → fixed orientation
- **Optional ground plane** with a soil texture so things don't float in space
- Scene template applies the asset's classification and respects orientation rules
- Scene preview uses the same lighting/preset as the bake (consistency from S-007)
- Existing grid stress test still available under a "Benchmark" tab for FPS measurement
- Scene template selection emits an analytics event

### Out of Scope

- Multi-asset scene composition (one type at a time for now)
- User-authored scene layouts beyond templates
- Day/night cycle or animated lighting
- Camera fly-throughs or animated previews
- Ground material editor

### Notes

- The scene preview is the main "demo for a designer colleague" surface. It needs to look credible at a glance, not benchmarky.
- Templates should be tunable too — count of instances, scatter density, spacing — exposed as controls in the scene preview panel.
- The trellis test from S-004 lives here too: set the trellis to `directional`, pick `hedge-row`, and verify it produces a believable row.
