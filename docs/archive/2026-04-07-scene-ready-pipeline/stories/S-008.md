## Workflow Consolidation + Information Architecture

### Context

The current toolbar exposes the implementation: `LODs (gltfpack)`, `LODs (Blender)`, `Billboard`, `Volumetric`, `Vol LODs`, `Production Asset`, `Reference Image`, `Test Lighting`. The user only knows what these mean because they designed the program. Plus there's a separate "Process" button in the left panel that the user has to remember to click first because of legacy cleanup behavior.

This story is the IA pass that lands at the end of the epic, after Tracks A-F have given us the actual functionality. It consolidates the per-step buttons into a single goal-oriented action: **"Prepare for scene"** — one click that runs the full pipeline (cleanup → classify → bake → LODs → manifest) using whatever settings/preset/profile the user has dialed in.

### Acceptance Criteria

- New primary action: **"Prepare for scene"** (or equivalent — name TBD with user)
  - Runs the full pipeline end-to-end with current settings
  - Output: a scene-ready bundle (production asset + LOD chain + manifest + thumbnails)
  - Single button, prominent placement
  - Progress indicator (which stage is running)
- Existing technique buttons (`Billboard`, `Volumetric`, `Vol LODs`, `Production Asset`, `LODs gltfpack`, `LODs Blender`) moved to an **"Advanced"** disclosure panel that's collapsed by default
- The "Process" left-panel button is either folded into "Prepare for scene" or removed if no longer needed
- **Label audit** — every button, label, panel header reviewed against the question "would a landscape designer who's never seen this app know what this does?" Rename anything that fails. Specific suspects:
  - "Production Asset" → something more descriptive
  - "Vol LODs" → spelled out
  - "VL0/VL1/VL2/VL3" → labeled with what they're for
  - The various "Generate" verbs → unified
- **Inline help text** on tuning controls (one-liner each, expanded tooltip optional)
- **First-run hint** — if no asset is loaded, show a brief explanation of what the app does and the recommended workflow
- The "10 click flow" the user described becomes a 2-3 click flow: upload → (optionally tune) → Prepare for scene → done
- Advanced users can still hit the technique buttons directly if they want
- Renames are recorded in an analytics event (so we can correlate UX changes with task completion times)

### Out of Scope

- Onboarding tutorial / guided tour
- A11y audit (worth doing, but follow-up)
- Mobile responsive layout
- Keyboard shortcuts
- Multi-language support

### Notes

- This is the most user-facing story in the epic. Worth a screenshot review with the user before merging.
- The label audit is small in lines of code but high in leverage. Don't skip it.
- Lands last because it depends on the actual functionality being in place. Trying to consolidate UI before S-005, S-006, S-007 ship would just rearrange broken things.
