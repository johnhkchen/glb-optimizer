## Context

T-009-01 bakes the tilted billboard. T-009-02 loads it and creates camera-facing instances. This ticket is the payoff: extend the existing 2-state visibility crossfade (horizontal billboard ↔ dome slices) into a 3-state crossfade (horizontal → tilted → dome slices), and wire the tilted bake into `generateProductionAsset` and `Prepare for scene` so it ships as part of the standard production bundle.

## Acceptance Criteria

### Three-stage crossfade math

Replace `updateBillboardVisibility` and `updateVolumetricVisibility` (or add new sibling functions) with a unified visibility function that handles three states based on `lookDownAmount = abs(camDir.dot(0,-1,0))`:

| `lookDownAmount` range | Horizontal | Tilted | Dome |
|------------------------|------------|--------|------|
| 0.00 - 0.30            | 1.0        | 0.0    | 0.0  |
| 0.30 - 0.55            | fade out   | fade in| 0.0  |
| 0.55 - 0.75            | 0.0        | 1.0    | 0.0  |
| 0.75 - 1.00            | 0.0        | fade out | fade in |

Use smoothstep for the fade curves. The fade band thresholds (`0.30`, `0.55`, `0.75`) should be settings fields so they can be tuned by eye:
- `tilted_fade_low_start` (default 0.30)
- `tilted_fade_low_end` (default 0.55)
- `tilted_fade_high_start` (default 0.75) — overlaps with the existing dome fade range
- The existing dome `fadeStart`/`fadeEnd` (currently 0.55/0.85 in `updateVolumetricVisibility`) are shifted to align with `tilted_fade_high_start = 0.75` and `tilted_fade_high_end = 1.0`

### Production bundle integration

- `generateProductionAsset(id)` runs the existing billboard + volumetric bakes AND the new tilted billboard bake. All three uploads happen sequentially
- `prepareForScene`'s "production" stage marks success only when all three artifacts exist
- `prepareForScene` orchestrator surfaces the tilted bake step in the per-stage progress UI (under the existing "production" bundle stage — substages are fine)
- The "Production" hybrid preview mode (`runProductionStressTest`) loads all three GLBs in parallel and instantiates billboard + tilted + volumetric at the same positions, all with `volumetricHybridFade = true` (or equivalent — rename if it makes the 3-state version clearer)

### Settings + UI

- Tuning panel exposes `tilted_fade_low_start`, `tilted_fade_low_end`, `tilted_fade_high_start` as sliders (range 0.0-1.0)
- Defaults are tuned by eye against the rose asset to look smooth
- Each slider fires a `setting_changed` analytics event (free via auto-instrumentation)

### Verification

- Manual: load the rose, run Prepare for scene, switch to Production preview, run stress test at count=20, slowly orbit from horizontal to overhead. The transition through the ~45° band should look noticeably smoother than the current 2-state crossfade — no "tipping over" billboards, no "paper-thin slices"
- The tilted variant adds ~300 KB to the production bundle size (acceptance: bundle size increase < 500 KB on the rose)

## Out of Scope

- Adaptive crossfade thresholds based on shape category
- Multiple tilt elevations (the bake is single-elevation per T-009-01)
- Removing the existing horizontal billboards (additive change)
- Custom shader-based blending (the alpha-multiplier crossfade is the v1)

## First-Pass Scope

This is the user-visible payoff ticket. Acceptable to spend a little extra time getting the default fade thresholds right against the rose asset, since the user will see the result first thing.
