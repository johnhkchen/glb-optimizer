## Context

The classifier from T-004-02 tells us what shape an asset is. This ticket builds the router that maps a shape category to a slicing/orientation strategy. The router is what makes the classification *useful* — without it, we just have labels and no behavior change.

## Acceptance Criteria

- New module / function `getStrategyForCategory(category)` returns a strategy config:
  - `slice_axis` — which principal axis to slice along (Y for round-bush, perpendicular-to-thin-axis for planar, etc.)
  - `slice_count` — default count for this category
  - `slice_distribution_mode` — `equal-height` / `vertex-quantile` / `visual-density`
  - `instance_orientation_rule` — `random-y` / `fixed` / `aligned-to-row` / `random-jitter`
  - `default_budget_priority` — for the scene budget system
- Strategy mappings for the starter taxonomy:
  - `round-bush` → slice along Y, visual-density, random-y orientation (matches current rose behavior)
  - `directional` → slice perpendicular to longest horizontal axis, equal-height, fixed orientation
  - `tall-narrow` → slice along Y with more layers, equal-height, random-y orientation
  - `planar` → slice perpendicular to thin axis, equal-height, aligned-to-row orientation
  - `hard-surface` → routes to the parametric pipeline from S-001 (no slicing)
- The bake functions from S-005 read the strategy from the asset's classified category and use those defaults (user can override per-setting)
- Strategy resolution emits a `strategy_selected` analytics event with `{category, strategy}`
- Manual verification: classify a directional asset, run the bake, confirm the slice axis is perpendicular to the long horizontal axis (not Y as the default would be)

## Out of Scope

- Multi-strategy comparison UI (T-004-04)
- Trellis-specific instance orientation in scene preview (S-006)
- New shape categories beyond the starter taxonomy

## First-Pass Scope

The router is a lookup table — keep it simple. Any per-category tuning that needs to be more granular can be exposed via the settings UI (the user overrides the strategy defaults if they want).
