## Context

Final story of the epic. T-008-01 gave us a primary action. T-008-02 hid clutter and renamed labels. This ticket adds the small explanatory touches that make the app self-explanatory for a first-time user — inline help on tuning controls and a brief first-run hint when the app loads with no assets.

## Acceptance Criteria

- Each tuning panel control gets a one-line help description:
  - Either inline (small italic text below the label)
  - OR a `?` tooltip on hover
  - Pick one pattern and use it consistently
- First-run hint: if no assets are loaded, the preview placeholder shows a brief "Get started" message:
  - Drop a `.glb` file or browse
  - Click "Prepare for scene" once it's loaded
  - Tune from the right panel if needed
  - Mark as accepted when it looks good
- The hint disappears once any asset has been uploaded in the current session
- Help text descriptions live in a single file (`static/help_text.js` or equivalent) so they're easy to edit / translate later
- Optional: a `?` icon next to each tuning section header that links to a longer explanation in `docs/`
- Manual verification: open the app fresh, see the hint; upload an asset, hint goes away; hover a tuning control, see help text

## Out of Scope

- Onboarding tutorial / guided tour
- Multi-language support
- Video walkthroughs
- A11y audit
- Keyboard shortcut hints

## First-Pass Scope

Final touches. Don't over-design. One line of help per control + one paragraph of first-run hint is enough. Anything more is scope creep.

## End of Epic

This is the last ticket in E-001. After it lands, the epic is done and the workflow transformation from "10 confusing clicks to remember" → "upload, click Prepare for scene, tune by eye, accept" is complete.
