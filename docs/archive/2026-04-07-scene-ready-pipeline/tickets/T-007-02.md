## Context

Today the bake (offscreen render) and the live preview (main scene) have separate light setups with different intensities. This is the source of "the calibration works in preview but not in production assets" frustration. This ticket collapses that into a single preset that drives both surfaces consistently — pick `midday-sun` once and both the bake AND the live preview use the same lighting.

## Acceptance Criteria

- The main scene's lights (in `initThreeJS`) read their intensities/colors from the current preset's `preview_config`
- The bake's `setupBakeLights` reads its intensities/colors from the current preset's `bake_config`
- Switching the `lighting_preset` setting:
  - Immediately re-applies lighting to the main scene
  - Marks the asset as "needs rebake" (a small UI hint, since the existing baked textures still reflect the old preset)
- The `bake_config` and `preview_config` for each preset should produce *visually similar* lighting on the same model — not pixel-identical (different camera positions, different render paths) but recognizably consistent
- Reference image calibration mode (from T-005-03) still works alongside presets — when active, it tints the current preset's colors
- Manual verification: with the rose loaded, switch presets and watch the live preview update; regenerate production asset and confirm the bake matches

## Out of Scope

- Reference image as a preset (T-007-03)
- Auto-rebake on preset change (just mark as stale)
- Real-time lighting in the live preview during bake (the live preview can update independently)
- Sky-as-background rendering

## First-Pass Scope

The tricky part is getting the bake_config and preview_config tuned to match. Don't aim for perfect parity — close enough that the user can tell which preset is active in both views.
