## Context

T-002-01 built the settings schema and backend persistence. This ticket does the inversion: every hardcoded constant in `static/app.js` that affects the bake or preview now reads from a settings object loaded from the backend. **No new UI** — that's T-002-03. The user-visible behavior must be identical when settings have their default values (regression-free inversion).

This is the meat of the foundational work. Get it right and every downstream tuning story becomes trivial — wire a slider to a setting field and you're done.

## Acceptance Criteria

- New JS module/section in `app.js` for settings management:
  - `currentSettings` global, populated from `/api/settings/:id` on file select
  - `loadSettings(id)` async fetch + assign + return
  - `saveSettings(id)` PUT current settings, debounced (used later by the UI)
  - `applyDefaults()` reset to schema defaults
- Replace these hardcoded constants/values throughout `app.js` with reads from `currentSettings`:
  - `VOLUMETRIC_LAYERS`, `VOLUMETRIC_RESOLUTION`
  - All `domeHeight = layerThickness * 0.5` style literals
  - `toneMappingExposure = 1.0` (renderer + offRenderers)
  - All `AmbientLight` intensities in `setupBakeLights` and the slice render
  - All `DirectionalLight` intensities in bake setups
  - `HemisphereLight` intensity
  - `envMapIntensity = 1.2` in `cloneModelForBake`
  - `alphaTest` values in volumetric/billboard materials
- The bake functions (`renderBillboardAngle`, `renderBillboardTopDown`, `renderLayerTopDown`, `setupBakeLights`, `cloneModelForBake`) read from `currentSettings` directly OR accept it as a parameter — pick one pattern and use it consistently
- `selectFile()` calls `loadSettings()` before `loadModel()` so settings are in place when bake/preview functions run
- File switch resets `currentSettings` to the new file's saved values (or defaults if none)
- **Regression check**: bake the rose with default settings, compare visually to a pre-change bake. They should be indistinguishable. Document the comparison approach in the work artifacts.
- **No new sliders, no new buttons.** This ticket is purely an architectural inversion. The UI lands in T-002-03.

## Out of Scope

- Tuning panel UI (T-002-03)
- New parameters not currently hardcoded (S-005)
- Live re-render on settings change (T-002-03)
- Settings UI feedback / unsaved indicators (T-002-03)
- Analytics events on settings changes (S-003)
