## Context

T-005-01 added the algorithmic improvements but only the slice mode and dome height are in the tuning panel. This ticket exposes the rest of the bake quality knobs (light intensities, exposure, env intensity) in the tuning UI from T-002-03 — they're already in the settings schema, just need controls in the panel.

This is the ticket where the user actually gets the "tune by eye" workflow they've been asking for. Every slider here also fires an analytics event for free via T-003-02's auto-instrumentation.

## Acceptance Criteria

- Tuning panel from T-002-03 gains controls for the bake quality settings that landed in T-002-01:
  - `bake_exposure` — slider 0.5 to 2.5 (already in T-002-03 schema, confirm wired)
  - `ambient_intensity` — slider 0 to 2.0
  - `hemisphere_intensity` — slider 0 to 2.0
  - `key_light_intensity` — slider 0 to 3.0
  - `bottom_fill_intensity` — slider 0 to 1.5
  - `env_map_intensity` — slider 0 to 3.0
  - `slice_distribution_mode` — dropdown (from T-005-01)
  - `dome_height_factor` — slider 0 to 1.0 (from T-005-01)
  - `ground_align` — checkbox (from T-005-01)
- Default values produce a noticeably better-shaded bake than current — leaf-to-leaf contrast visible without going pitch black on the underside
- Each control has a one-line label and shows its current value
- Changing any control triggers a `setting_changed` analytics event (free via T-003-02)
- Visual indicator on the asset (in the file list) when settings differ from defaults
- Manual verification: load the rose, bump `key_light_intensity` to 2.0 and `dome_height_factor` to 0.7, regenerate production asset, see visible improvement

## Out of Scope

- Color calibration mode (T-005-03)
- Lighting presets (S-007)
- Live re-render on slider drag (regenerate-on-action is fine)
- Reset-to-defaults already covered in T-002-03

## First-Pass Scope

Wire the existing settings to UI controls. No need for fancy presets, no need for tooltips beyond labels, no need to be pretty. Functional is enough.
