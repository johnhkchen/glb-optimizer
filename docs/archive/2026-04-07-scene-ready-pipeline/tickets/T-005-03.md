## Context

The reference image calibration we built in the prior epic exists but is bolted on as a one-shot button. This ticket makes it a proper setting: a `color_calibration_mode` enum that the user picks in the tuning panel. The full preset enum lands in S-007 — for now we just need `none` and `from-reference-image`.

This sets the stage for S-007 where the calibration system gets absorbed into the lighting preset framework alongside named presets like `midday-sun`.

## Acceptance Criteria

- New settings field: `color_calibration_mode` enum
  - `none` — neutral white lighting (default)
  - `from-reference-image` — use the existing palette extraction from the reference image
- New settings field: `reference_image_path` (string, optional) — path to the asset's reference image if uploaded
- Tuning panel adds a dropdown for `color_calibration_mode`
- When set to `from-reference-image`, the tuning panel shows the existing "Reference Image" upload control (or directs the user to it)
- When set to `none`, calibration is bypassed and bake uses neutral lighting
- The mode persists per-asset via the settings system
- Mode changes emit `setting_changed` analytics events
- Manual verification: load rose, set mode to `from-reference-image`, upload the rose photo, regenerate, verify the bake reflects the calibration; switch back to `none`, regenerate, verify it goes back to neutral

## Out of Scope

- Lighting presets enum (S-007)
- Improving the palette extraction (kept as-is for now)
- New calibration sources (HDR images, color picker, etc.)

## First-Pass Scope

This is a wiring ticket: existing functionality plumbed through the settings system. No new calibration techniques, no new UI design — just making the existing reference image flow into a proper setting.
