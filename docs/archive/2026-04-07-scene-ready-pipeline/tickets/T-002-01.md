## Context

Foundation for all of E-001. Today every parameter that affects the bake (slice count, dome height, exposure, ambient/key/hemisphere intensities, env map intensity, alpha test thresholds, color calibration mode) is a hardcoded constant in `static/app.js`. To unblock S-002's UI and every downstream tuning story, we first need a versioned per-asset settings object with backend persistence.

This ticket defines the schema, builds the storage layer, and exposes read/write endpoints. **No client-side wiring yet** â€” that's T-002-02. **No UI yet** â€” that's T-002-03.

## Acceptance Criteria

- Document the schema in `docs/knowledge/settings-schema.md`:
  - `schema_version: 1`
  - All currently hardcoded constants enumerated with default values matching today's behavior
  - Field types, ranges, and one-line descriptions
  - Migration policy (how v2 will handle v1 files)
- New Go file `settings.go` with:
  - `AssetSettings` struct mirroring the schema
  - `DefaultSettings()` returning the canonical defaults
  - `LoadSettings(id, dir)` reading from `~/.glb-optimizer/settings/{id}.json`, returning defaults if missing
  - `SaveSettings(id, dir, settings)` writing atomically (temp file + rename)
  - JSON marshaling with stable field order
- New HTTP handlers in `handlers.go`:
  - `GET /api/settings/:id` returns the asset's settings (or defaults if none saved)
  - `PUT /api/settings/:id` accepts a JSON body, validates it, writes to disk, returns 200
  - Validation: schema_version must match, numeric ranges must hold
- Routes registered in `main.go`
- Settings directory created at startup if missing
- `FileRecord` gets a `HasSavedSettings bool` field (computed from disk presence) so the file list can show a tuning indicator later
- Unit-style verification: write defaults, read back, assert equality (a Go test file is fine if there's existing test infra; otherwise document a manual verification recipe)
- Schema covers at least these fields (others can be added in T-002-02 if needed):
  - `volumetric_layers` (int, default: current adaptive base of 4)
  - `volumetric_resolution` (int, default: 512)
  - `dome_height_factor` (float, default: 0.5)
  - `bake_exposure` (float, default: 1.0)
  - `ambient_intensity` (float, default: 0.5)
  - `hemisphere_intensity` (float, default: 1.0)
  - `key_light_intensity` (float, default: 1.4)
  - `bottom_fill_intensity` (float, default: 0.4)
  - `env_map_intensity` (float, default: 1.2)
  - `alpha_test` (float, default: 0.15)
  - `lighting_preset` (string enum, default: "default" â€” preset enum lands in S-007)
