## Context

With both hard-surface reconstruction and organic distillation proven, the optimizer needs a budget-aware system that allocates triangle and texture budgets per asset based on scene role (hero, mid-ground, background) and asset type (constructed vs organic). This is the convergence point that makes the optimizer a real pipeline stage rather than a collection of individual techniques.

## Acceptance Criteria

- Define a scene budget schema (total triangle budget, total texture memory budget)
- Support per-asset budget allocation based on:
  - Asset type (hard-surface vs organic — determines distillation strategy)
  - Scene role (hero / mid-ground / background — determines LOD level and budget share)
- Given a set of assets + budget, automatically select and apply the appropriate distillation strategy
- Demonstrate with a sample scene: 1 raised bed + 3 rose bushes at different distances, total budget <50K triangles
- Output a manifest (JSON) describing each asset's representation, triangle count, texture size, and LOD chain
- API endpoint: POST /api/optimize-scene with assets + budget, returns optimized bundle
