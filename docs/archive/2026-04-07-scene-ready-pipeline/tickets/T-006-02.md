## Context

T-006-01 built the framework. This ticket implements the actual templates a designer would use AND adds the UI for picking between them. After this lands, the user can switch between "row of trellises," "mixed bed," and "rock garden" layouts to evaluate how an asset reads in context.

Also adds an optional ground plane so things don't float in space.

## Acceptance Criteria

- Implemented templates:
  - `hedge-row` — single row, evenly spaced, fixed orientation (showcases directional assets)
  - `mixed-bed` — natural scatter with size variation 0.85-1.15× (showcases round bushes)
  - `rock-garden` — sparse scatter, larger size variation 0.7-1.3×
  - `container` — small tight cluster of 5-10 instances
  - `grid` — current 10×10 grid behavior, kept for FPS benchmarking
- New scene preview UI section in the toolbar (or stress test panel):
  - Template dropdown
  - Instance count input
  - "Run scene" button
  - Toggle for ground plane
- Ground plane: a simple textured plane at Y=0 with a soil-like color (no texture asset needed for v1, just a brown base color)
- Template choice + instance count + ground plane toggle stored in `currentSettings` so they persist per asset
- Template change emits a `scene_template_selected` analytics event
- The scene preview uses the same lighting preset as the bake (consistency from S-007)
- Manual verification: load rose, pick `mixed-bed`, run, see scattered varied bushes; switch to `grid`, run, see the original benchmarking grid
- Trellis test from T-004-05 lives here too: load trellis, pick `hedge-row`, see a believable row

## Out of Scope

- User-authored layouts beyond the templates
- Multi-asset scenes (one type at a time)
- Day/night cycle / animated lighting
- Camera fly-throughs
- Ground material editor (just a base color is fine)

## First-Pass Scope

Templates need to be functional and visually distinct. The ground plane is a nice-to-have — a flat colored quad is enough; no need for a fancy material.
