## Context

T-002-01 built the schema and persistence. T-002-02 inverted the codebase to read from settings. This ticket adds the UI skeleton — sliders and dropdowns that let the user tune parameters by eye and have the changes persist per-asset.

The UI is intentionally minimal: a new "Tuning" section in the right panel (alongside the existing Mesh/Texture/Output settings) with one control per setting field. No fancy presets, no profiles, no inline help. That comes in S-005, S-007, S-008.

## Acceptance Criteria

- New "Tuning" section in `static/index.html`'s right panel, below the existing settings sections
- One control per `AssetSettings` field from T-002-01's schema:
  - `volumetric_layers` → number input (range 1-12)
  - `volumetric_resolution` → dropdown (256, 512, 1024)
  - `dome_height_factor` → slider 0.0 - 1.0
  - `bake_exposure` → slider 0.5 - 2.5
  - `ambient_intensity` → slider 0.0 - 2.0
  - `hemisphere_intensity` → slider 0.0 - 2.0
  - `key_light_intensity` → slider 0.0 - 3.0
  - `bottom_fill_intensity` → slider 0.0 - 1.5
  - `env_map_intensity` → slider 0.0 - 3.0
  - `alpha_test` → slider 0.0 - 0.5
  - `lighting_preset` → dropdown (only "default" for now; S-007 fills it in)
- Each control shows its current value next to it (live updates as user drags)
- Selecting a file populates the controls from `currentSettings`
- Changing a control updates `currentSettings` and triggers a debounced PUT to the backend (300ms)
- "Reset to defaults" button fetches schema defaults and applies them
- The user can re-bake the asset after tuning by clicking the existing "Production Asset" button (or any other generation action) — the bake reads `currentSettings` per T-002-02
- Visual indicator (subtle, e.g., a dot next to the section header) when settings differ from defaults — helps the user notice they have unsaved tuning
- **Live preview is NOT required** — clicking generate/bake to see the result is fine for this skeleton ticket. Real-time preview as you drag is a future improvement.
- Existing presets ("Max Quality", "Balanced", etc.) and Mesh/Texture/Output sections are unchanged
- Layout doesn't break the existing right panel — Tuning section scrolls with the rest

## Out of Scope

- Named profiles, save/load profiles (S-003)
- Inline help text or tooltips (S-008)
- Live re-render on slider drag
- Lighting preset enum beyond "default" (S-007)
- New tunable parameters beyond what's in the schema (S-005)
- Unsaved-changes warning when navigating away
- Undo / history

## Notes

- Match the existing right-panel CSS conventions (`.setting-row`, `.range-value`, etc.) — don't introduce a new visual style.
- The debounced save is important — don't PUT on every pixel of slider drag.
- This is a skeleton, not a polished UI. Functional > pretty for this ticket.
