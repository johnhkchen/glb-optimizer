## Context

The current toolbar has 7+ technique buttons (`LODs gltfpack`, `LODs Blender`, `Billboard`, `Volumetric`, `Vol LODs`, `Production Asset`, `Test Lighting`, `Reference Image`) plus a separate `Process` button in the left panel. The user has to remember which to click and in what order. This ticket replaces all of that with a single goal-oriented primary action that runs the full pipeline end-to-end.

The technique buttons stay in the codebase (advanced users can still hit them) but get moved behind an "Advanced" disclosure in T-008-02.

## Acceptance Criteria

- New primary action button: **"Prepare for scene"** (placement: prominent, top of toolbar or in left panel)
- Clicking runs the full pipeline in order, using current settings:
  1. (If not already done) Run gltfpack cleanup pass
  2. Classify shape (S-004)
  3. Generate LODs
  4. Generate production asset (billboard + volumetric per shape category)
  5. Optionally generate scene preview thumbnail
- Progress indicator showing current stage
- On completion: show the result (the production asset bundle) and surface a "View in scene" action that runs the user's last scene template
- If any stage fails, the error is shown clearly and the pipeline stops at that stage (user can fix and retry)
- The action emits a `prepare_for_scene` analytics event with `{stages_run, total_duration_ms, success}`
- The existing technique buttons still work in this ticket (they get hidden in T-008-02)
- Manual verification: upload a fresh rose, click Prepare for scene, watch progress, end up with a production asset ready to view in a scene template

## Out of Scope

- Advanced disclosure UI (T-008-02)
- Label rename (T-008-02)
- Inline help (T-008-03)
- Pipeline retry / resume from failure
- Background pipeline execution

## First-Pass Scope

The orchestrator just calls the existing functions in sequence. No need for elaborate state machines or rollback. If it works in the happy path and shows clear errors on failure, it's done.
