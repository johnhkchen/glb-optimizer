## Context

S-005 exposed granular bake controls. This ticket consolidates those controls into named presets that map to the concepts a landscape designer actually thinks in: "midday sun," "overcast," "golden hour," etc.

A preset is a named bundle of light intensities, colors, env map gradient stops, and tone mapping exposure. Picking a preset overwrites those settings. Users can still tune from there; the preset is just the starting point.

## Acceptance Criteria

- New schema definition in `static/presets/lighting.js` (or equivalent JS module):
  - Each preset has: `id`, `name` (display), `description` (one line), `bake_config`, `preview_config`
  - `bake_config`: `{ambient, hemisphere_sky, hemisphere_ground, key_intensity, fill_intensity, env_gradient: [color, color, color], tone_exposure}`
  - `preview_config`: same structure (used for the live preview scene; T-007-02 wires consistency)
- Initial preset set:
  - `default` — current neutral baseline (matches T-002-01 defaults)
  - `midday-sun` — bright neutral white, strong key, mild tone exposure
  - `overcast` — soft, low-contrast, slightly cool
  - `golden-hour` — warm key colors, low ambient, dramatic
  - `dusk` — cool blue ambient, low key intensity
  - `indoor` — soft fill, no strong key
- Existing `lighting_preset` settings field (added in T-002-01 schema) becomes the discriminator — selecting a preset rewrites the dependent settings (ambient, key, hemisphere, etc.) to that preset's `bake_config`
- New backend or frontend module exposing `getPreset(id)` that returns the preset definition
- Manual verification: pick `golden-hour`, confirm the dependent settings update in the tuning panel, regenerate the bake, see warmer tones

## Out of Scope

- Bake-preview consistency wiring (T-007-02)
- Reference image as a preset option (T-007-03)
- Custom user-authored presets
- HDR environment maps
- Per-light editing inside a preset

## First-Pass Scope

Get the schema right and ship a starter set of 5-6 presets. Don't agonize over the exact color values — the user can override per-asset. Just make sure each preset is visibly distinct from the others.
