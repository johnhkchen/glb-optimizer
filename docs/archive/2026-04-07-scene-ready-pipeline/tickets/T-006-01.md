## Context

The current 100x stress test is a perfect grid of identical instances with identical Y rotations. It's useful for FPS measurement but useless for evaluating "does this asset look credible in a planted bed." This ticket builds the framework for scene templates: scattered placements with per-instance variation in scale, position jitter, and orientation.

The framework is the foundation; specific templates and ground plane land in T-006-02 and T-006-03.

## Acceptance Criteria

- New JS module `scene_templates.js` (or section in `app.js`) with:
  - `SceneTemplate` interface: `{id, name, generate(asset, count) → InstanceSpec[]}`
  - `InstanceSpec`: `{position: Vector3, rotation: Quaternion, scale: number}`
- New scatter helpers:
  - `scatterRandomly(bounds, count, seed)` — random points within bounds with min-distance constraint
  - `scatterInRow(start, end, count, jitter)` — line of instances with optional position jitter
  - `applyVariation(specs, scaleRange, jitterAmount)` — adds per-instance scale + position variation
- The scene preview reads the asset's `shape_category` (from S-004) and applies the right rotation rule:
  - `round-bush`, `tall-narrow` → random Y rotation per instance
  - `directional`, `planar` → fixed orientation (or small ±5° jitter)
  - `hard-surface` → fixed orientation
- A simple debug template `debug-scatter` that scatters 20 instances randomly with size variation, used to validate the framework
- Manual verification: load the rose, run debug-scatter, see scattered bushes with size variation and random rotation
- Existing grid stress test still works (kept as a "Benchmark" template)

## Out of Scope

- Specific scene templates beyond the debug scatter (T-006-02)
- Ground plane (T-006-03)
- UI for picking templates (T-006-02)
- Multi-asset scenes

## First-Pass Scope

Get the framework right. Templates and UI come next. The variation system should be deterministic per seed so tweaks are reproducible.
