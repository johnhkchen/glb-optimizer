## Context

Reference image calibration is currently a separate `color_calibration_mode` from S-005. This ticket folds it into the lighting preset enum so it's discoverable alongside the named presets. Users picking from one dropdown find both built-in presets ("midday-sun", "overcast") and custom calibration ("from-reference-image") in the same place.

## Acceptance Criteria

- New preset `from-reference-image` added to the preset enum from T-007-01
- When this preset is selected, the bake/preview lights are tinted by the palette extracted from the reference image (existing code from T-005-03)
- The tuning panel's lighting preset dropdown shows this option alongside the named presets, with a one-line description ("Calibrated from your reference image")
- Selecting this preset shows the reference image upload control inline (or surfaces the existing one)
- The `color_calibration_mode` setting from T-005-03 is removed/deprecated — its job is now the lighting preset
- Backwards compatibility: existing assets with `color_calibration_mode: from-reference-image` are auto-migrated to `lighting_preset: from-reference-image`
- Manual verification: load rose, set lighting preset to `from-reference-image`, upload rose photo, regenerate, see calibrated bake; switch to `midday-sun`, regenerate, see neutral white bake

## Out of Scope

- Improving the palette extraction itself
- Multiple calibration sources (HDR, color picker)
- Per-asset reference image library

## First-Pass Scope

Plumbing ticket. The reference image calibration code from S-005 doesn't change — it just gets reached through a different setting. The migration step is cosmetic since this is a single-user dev tool — for now, deleting the old field is fine if no real production data exists.
