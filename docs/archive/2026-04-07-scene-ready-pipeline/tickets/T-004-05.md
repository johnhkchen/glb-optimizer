## Context

The trellis case is the marquee test for everything S-004 built. This ticket verifies end-to-end that a trellised asset (peas, beans, climbing plants on a fence/lattice) flows through the pipeline correctly:

1. Uploaded → classifier returns `directional`
2. Strategy router uses perpendicular-to-thin-axis slicing + fixed orientation
3. Production asset bakes with those parameters
4. (Sneak preview of S-006) When placed in a row, instances stay aligned

If this works for a trellis, it works for the broader directional category — fences, espaliers, hedge rows, etc.

## Acceptance Criteria

- A trellis-class test asset is in the repo at `assets/` (sourced from TRELLIS or wherever — even a synthetic one is fine)
- Running the upload flow on the trellis asset:
  - Auto-classifier returns `directional` (or triggers comparison UI, where the user picks `directional`)
  - Strategy router selects perpendicular-axis slicing
  - Production asset bakes successfully
  - The baked output preserves the directional shape (slices don't cut the trellis into incoherent fragments)
- A simple stress test of 5 trellis instances in a row (using existing stress test, before S-006 lands):
  - All 5 instances have the same Y rotation (no per-instance random rotation)
  - The result reads as "a row of trellises" not "a chaotic mess"
- Documented in `docs/active/work/T-004-05/review.md` with screenshots
- Any classifier or router bugs found are fixed (this is the first real validation of the new code path)

## Out of Scope

- New shape categories
- Scene templates for trellis arrangements (S-006)
- Asymmetric scene placements (S-006)
- Branching topology

## First-Pass Scope

This is a validation ticket. The bar is "the trellis case works well enough to demo to a designer," not "perfect rendering." If the bake has artifacts but the orientation/classification is correct, that's a pass for this ticket — visual polish is a follow-up.
