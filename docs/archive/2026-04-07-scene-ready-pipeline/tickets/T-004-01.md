## Context

Research spike. Before committing to PCA-based shape classification as the unlock for the trellis problem and S-006's scene preview, we need to validate that naive PCA on a vertex cloud actually distinguishes the shape categories we care about. If it doesn't, we need to know the fallback (bounding-box ratios + density histogram, or something else entirely) before sinking effort into the implementation.

This is a research-only ticket. No production code. Output is a research artifact that informs T-004-02.

## Acceptance Criteria

- New script `scripts/spike_shape_pca.py` (Python with numpy + trimesh, or Go if preferred):
  - Loads a GLB
  - Extracts the vertex cloud (with world transforms applied)
  - Computes PCA: principal axes and eigenvalues
  - Reports: axis vectors, eigenvalue magnitudes, axis-magnitude ratios, aspect ratio, density histogram per principal axis
- Run the script on existing reference assets:
  - `assets/rose_julia_child.glb` — expected: round-bush
  - `assets/wood_raised_bed.glb` — expected: hard-surface (rectangular)
  - At least one trellis-class asset (find one online or generate one for testing if none exists in the repo)
- Document findings in `docs/active/work/T-004-01/research.md`:
  - PCA results table for each test asset
  - Whether axis-magnitude ratios meaningfully separate the categories
  - Suggested classification thresholds (or "this approach doesn't work cleanly, try X instead")
  - Confidence score formula proposal
  - Edge cases observed
- Recommendation: ship classifier as PCA-based, OR pivot to a different approach (and what)

## Out of Scope

- Production classifier code (T-004-02)
- Strategy router (T-004-03)
- Multi-strategy comparison UI (T-004-04)
- Performance optimization (the spike can be slow)

## First-Pass Scope

This is research, not engineering. The script can be ugly. The artifact is the deliverable — make sure it's clear enough that T-004-02 can reference it as the basis for the production classifier.

If a trellis asset isn't available, document that gap and use the existing hard-surface raised bed as a stand-in for the directional/asymmetric case.
