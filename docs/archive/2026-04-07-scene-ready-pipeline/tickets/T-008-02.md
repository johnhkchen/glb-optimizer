## Context

T-008-01 added "Prepare for scene" as the primary action. This ticket cleans up everything else: hides the technique buttons behind an "Advanced" disclosure, audits every label for clarity, and renames anything that fails the "would a landscape designer who's never seen this app know what this does?" test.

## Acceptance Criteria

- New collapsed "Advanced" disclosure section in the toolbar containing:
  - `Billboard`, `Volumetric`, `Vol LODs`, `Production Asset`
  - `LODs (gltfpack)`, `LODs (Blender)`
  - `Test Lighting`, `Reference Image` (if not already absorbed into the lighting preset from T-007-03)
- Disclosure is collapsed by default; user can expand if they want direct technique access
- Label audit + rename pass:
  - "Production Asset" → e.g. "Build hybrid impostor" or whatever reads better
  - "Vol LODs" → "Volumetric LOD chain"
  - "VL0/VL1/VL2/VL3" → "Volumetric (highest detail)" / "(medium)" / "(low)" / "(billboard)"
  - "Billboard" → "Camera-facing impostor"
  - All "Generate" verbs unified to one verb
  - Toolbar action labels reviewed for jargon
- Tuning panel section names and field labels reviewed (not just the toolbar)
- The "Process" left-panel button gets either folded into "Prepare for scene" or removed if no longer relevant — pick the simpler answer based on what cleanup it currently does
- All renames logged in a brief `docs/active/work/T-008-02/review.md` so future contributors can find old names
- Manual verification: a teammate (or fresh-eyes review) goes through the toolbar and confirms each button is understandable without context

## Out of Scope

- Inline help text (T-008-03)
- First-run hint (T-008-03)
- A11y audit
- Mobile responsive layout
- Keyboard shortcuts

## First-Pass Scope

Renames are cheap, high-leverage. Don't agonize over the perfect label — pick something better than the current and move on. The label log lets future contributors propose better.
