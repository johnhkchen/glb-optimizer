## Context

The current bake has three known visual problems:

1. **Dome curvature is gone** — vertex-quantile slicing flattened the umbrella shape we had earlier
2. **Ground clipping** — bottom slice's actual floor sits below Y=0, leaves clip into the ground in scene previews
3. **Center-of-mass drag** — vertex-quantile boundaries pull the visual top of the plant down because dense lower foliage gets more boundaries than sparse upper foliage

This ticket addresses all three by adding a new slice distribution mode (`visual-density`) and exposing the controls through the settings system from S-002. The dome height and slice mode become tunable per asset.

## Acceptance Criteria

- New slice distribution mode `visual-density` in `renderHorizontalLayerGLB`:
  - Filters vertices that are part of the trunk/stem (heuristic: bottom 10% of bounding box height) before computing quantile boundaries
  - Optionally weights remaining vertices by distance from the center vertical axis (foliage out at the edges counts more than dense central stems)
  - Result: boundaries reflect *visible* foliage, not raw geometry
- New settings field: `slice_distribution_mode` enum with values `equal-height`, `vertex-quantile`, `visual-density` (default: `visual-density`)
- New settings field: `dome_height_factor` is already in the schema from T-002-01 — wire it through `createDomeGeometry` (currently hardcoded at 0.5 of layer thickness)
- Ground alignment fix: when generating the volumetric GLB, apply a Y offset so the bottom slice's lowest vertex sits exactly at Y=0
- Ground alignment is also a settings field: `ground_align` boolean (default: true)
- Backwards compatibility: `equal-height` mode reproduces the original simple horizontal slicing for assets that need it
- Manual verification: re-bake the rose with `visual-density` mode + `dome_height_factor: 0.7` and confirm the silhouette has more curvature than the current pancake stack
- All three new settings emit `setting_changed` analytics events when modified

## Out of Scope

- UI sliders for these new settings (T-005-02 surfaces them in the tuning panel)
- Lighting changes (T-005-02)
- Color calibration (T-005-03)
- Per-slice customization

## First-Pass Scope

`visual-density` mode is the algorithmic addition. Don't over-engineer the heuristic — the trunk filter + radial weight is enough for v1. We can iterate based on what the analytics show users actually overriding.
