## Context

First brick of S-009. Generalize the existing `renderBillboardAngle` to accept a camera elevation parameter, add a `renderTiltedBillboardGLB` that bakes N angles at the new elevation, and wire up server-side storage + endpoint that mirrors the existing billboard upload path.

This is the bake side only — runtime loading and crossfade are T-009-02 and T-009-03.

## Acceptance Criteria

- `renderBillboardAngle(model, angleRad, resolution)` → `renderBillboardAngle(model, angleRad, resolution, elevationRad = 0)`
  - `elevationRad = 0` reproduces existing behavior bit-for-bit (regression-free)
  - When `elevationRad > 0`, camera is positioned at angle (azimuth) AND elevated above the model's Y center by `dist * sin(elevationRad)`, looking at the model's center
  - The captured quad's `quadHeight` accounts for the elevated viewing angle so the rendered image fits the quad without stretching
- New JS function `renderTiltedBillboardGLB(model, numAngles, elevationRad, resolution)` parallels `renderMultiAngleBillboardGLB` but uses the elevated camera. Default constants: `TILTED_BILLBOARD_ANGLES = 6`, `TILTED_BILLBOARD_ELEVATION_RAD = Math.PI / 6` (30°), `TILTED_BILLBOARD_RESOLUTION = 512`
- New `generateTiltedBillboard(id)` async function that bakes the GLB and POSTs it to a new endpoint
- New backend endpoint `POST /api/upload-billboard-tilted/:id` that stores `{outputs}/{id}_billboard_tilted.glb`
- New `FileRecord.HasBillboardTilted bool` field (omitempty)
- `scanExistingFiles` detects `_billboard_tilted.glb` on startup and sets the flag
- `handleDeleteFile` removes the new file when an asset is deleted (closes the symmetric-cleanup gap previously flagged)
- `handlePreview` accepts `version=billboard-tilted` and serves the file
- Manual verification: bake a rose with the new function via devtools (`generateTiltedBillboard(selectedFileId)`), confirm the GLB lands on disk and `/api/preview/:id?version=billboard-tilted` serves it

## Out of Scope

- Loading the GLB into the live preview (T-009-02)
- Instance creation / camera-facing runtime behavior (T-009-02)
- Three-stage crossfade math (T-009-03)
- Including the tilted bake in `Prepare for scene` (T-009-03)
- Toolbar button for direct invocation (the function is callable from devtools for now)
- Multiple tilt elevations / per-angle tilt tuning
- Settings UI exposing the elevation as a slider

## First-Pass Scope

The bake side should be a near-mechanical extension of the existing billboard path. Resist the urge to refactor `renderBillboardAngle` beyond adding the elevation parameter. The new constants live near the existing `BILLBOARD_ANGLES` constant block.
