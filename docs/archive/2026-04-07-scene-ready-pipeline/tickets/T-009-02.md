## Context

T-009-01 bakes the tilted billboard GLB and stores it server-side. This ticket loads it on the client, creates instanced meshes from it, and adds a preview button so the user can view it in isolation. No crossfade yet — that's T-009-03.

The instance creation reuses the existing `createBillboardInstances` pattern: read N angle quads from the GLB, partition the instance positions across them with a seeded variant assignment, build one InstancedMesh per variant, and update Y rotation each frame to face the camera.

## Acceptance Criteria

- New `createTiltedBillboardInstances(model, positions)` function that mirrors `createBillboardInstances`:
  - Parses quads named `tilted_billboard_${i}` from the loaded GLB (the bake function should use this naming convention from T-009-01)
  - Builds one InstancedMesh per variant
  - Each instance gets a random angle variant (seeded by index) for visual variety
  - Instances are camera-facing in yaw (azimuth) — same `Math.atan2(camPos.x - pos.x, camPos.z - pos.z)` rotation as the horizontal billboards
  - Tracked in a new `tiltedBillboardInstances[]` array (mirroring `billboardInstances`)
- New `updateTiltedBillboardFacing()` function that updates the per-instance yaw each frame
- The animate loop calls `updateTiltedBillboardFacing()` when there are tilted instances active (mirroring the existing `updateBillboardFacing()` call)
- `clearStressInstances` resets `tiltedBillboardInstances = []`
- New `data-lod="billboard-tilted"` button in the LOD toggle bar with label "Tilted" (or similar — name TBD with user during ticket execution)
- Preview button enabled when `file.has_billboard_tilted` is true
- Selecting the "Tilted" preview loads `/api/preview/:id?version=billboard-tilted` and creates instances at the asset's position
- Manual verification: bake a tilted billboard via T-009-01, click the new "Tilted" preview button, confirm the bushes appear elevated/tilted in the preview and rotate to face the camera as you orbit

## Out of Scope

- Three-stage crossfade with horizontal billboards and dome slices (T-009-03)
- Including the tilted bake in `Prepare for scene` (T-009-03)
- Settings UI for the bake elevation
- Per-instance tilt jitter

## First-Pass Scope

This is plumbing. Mirror the existing billboard loader code path; do not try to share more than necessary between the two. Both will continue to exist in parallel.
